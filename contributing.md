# Contributing

Thanks for your interest in contributing!

## Getting started

- Fork the repository and create a feature branch.
- Run tests locally before opening a PR:
  - `dart test`

## Pull requests

- Keep changes focused and well-scoped.
- Include tests for bug fixes or new features when applicable.
- Update documentation if behavior changes.

## Code style

- Follow the project's Dart lint rules.
- Prefer small, readable functions and clear names.

## Reporting issues

- Use the issue tracker and include reproduction steps.
- Provide sample FIT files when possible.
